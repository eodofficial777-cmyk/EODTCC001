# 如何將您的專案程式碼推送到 GitHub

這份文件將引導您如何將您在 Firebase Studio 中的專案程式碼，完整地推送到您指定的 GitHub 倉庫。

請在您的本機電腦上，打開一個終端機 (Terminal) 或命令提示字元 (Command Prompt)，並按照以下步驟操作。

**前提：** 您需要在您的電腦上安裝 [Git](https://git-scm.com/downloads)。

---

## 步驟一：進入您的專案資料夾

首先，您需要將在 Firebase Studio 中的專案檔案下載到您的電腦上，並解壓縮。然後使用終端機進入那個資料夾。

```bash
# 將 "您的專案資料夾路徑" 替換成您實際的資料夾路徑
cd "您的專案資料夾路徑"
```

---

## 步驟二：初始化 Git 倉庫

如果這是您**第一次**在這個資料夾中使用 Git，您需要先初始化一個本地的 Git 倉庫。

```bash
git init
```

> **提示：** 這個指令只需要在專案一開始執行一次。如果資料夾內已經有 `.git` 這個隱藏資料夾，代表已經初始化過了，可以跳過此步驟。

---

## 步驟三：連接到您的遠端 GitHub 倉庫

接下來，我們需要告訴本地的 Git 倉庫，要將程式碼推送到哪個網路上的位置。

```bash
git remote add origin https://github.com/eodofficial777-cmyk/EODTCC001.git
```

> **提示：** `origin` 是遠端倉庫的預設名稱。這個指令也只需要在專案一開始執行一次。如果您之前已經設定過但想更換，可以先執行 `git remote remove origin` 再重新執行此指令。

---

## 步驟四：加入所有變更

這個指令會將資料夾中所有新增、修改的檔案都加入到 Git 的「暫存區」，準備進行提交。

```bash
git add .
```

> **提示：** `.` 代表目前資料夾下的所有檔案。

---

## 步驟五：提交變更

這個指令會將暫存區的所有內容，建立一個「提交紀錄」(commit)。您可以把 `-m` 後面的訊息想像成這次存檔的摘要或筆記。

```bash
git commit -m "在這裡寫下這次更新的摘要，例如：實作獎勵發放系統"
```

> **提示：** 良好的提交訊息有助於您未來追蹤每次的變更內容。

---

## 步驟六：推送到 GitHub

最後，這個指令會將您剛剛提交的所有內容，從您的電腦正式推送到 GitHub 上的 `main` (或 `master`) 分支。

```bash
git push -u origin main
```

> **提示：**
> *   第一次推送時，使用 `-u` 參數可以將本地的 `main` 分支與遠端的 `origin/main` 分支關聯起來。之後您只需要執行 `git push` 即可。
> *   如果您的預設分支名稱是 `master`，請將指令中的 `main` 改為 `master`。
> *   執行此指令時，終端機可能會要求您輸入 GitHub 的帳號和密碼（或是 Personal Access Token）。

---

恭喜您！完成以上步驟後，您的所有程式碼就成功地被推送到 GitHub 上了。未來您在 Firebase Studio 中有任何更新，只需要重複**步驟四到六**即可。
